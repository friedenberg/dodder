# repo_configs

Repository-level configuration management and overlays.

## Key Types

- `Config`: Main repository configuration with defaults and options
- `ConfigOverlay`: Layered configuration interface
- `ConfigOverlay2`: Extended overlay with blob store ID support
- `Defaults`: Default type and tags for new objects
- `TypedBlob`: Type-safe configuration blob wrapper

## Versions

- `V0`, `V1`, `V2`: Versioned configuration formats
- `DefaultsV1`: Default values structure

## Features

- Default object type and tags
- File extension configuration
- Print options (colors, formatting)
- Tool options (merge tool configuration)
- Overlay system for configuration composition
- Default blob store ID management
